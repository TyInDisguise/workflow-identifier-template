# As-Is Workflow Record Schema

Every workflow is documented against this template. Every field must be populated OR explicitly marked "n/a" with a reason.

## 1. Identification

- Name (verb + object, e.g. "Process new customer order")
- Owner (accountable party)
- Performer(s) (who actually executes day-to-day)
- Date documented
- Interviewee (name, role)

## 2. Trigger & Volume

- Trigger (what event/condition starts a run)
- Initiator (person, system, inbound message, time-based)
- Frequency (per day / week / month)
- Volume (typical instances per period)
- Duration (avg minutes per instance)

## 3. Inputs

- Data (fields, attachments, required info)
- Source systems (CRM, email, spreadsheet, etc.)
- Received from (person or system)

## 4. Steps

Numbered list. Each step:
- Action (verb-first description)
- Tool/system used
- Typical duration

## 5. Decision Rules

Explicit if/then logic — no "it depends."
- If [condition] → [action]
- If [condition] → [action]

## 6. Exceptions

- What breaks this workflow
- Approximate frequency of each (%)
- How each is handled

## 7. Outputs

- Deliverable (document, record update, email)
- Destination (where / who)
- Format

## 8. Success Criteria

- How "done right" is judged
- Who verifies, when

## 9. Pain Points

At least one in the performer's own words.
- Verbatim quote(s)
- Most frustrating aspect
- Where they wait or context-switch
- First thing they'd fix and why

## 10. Notes

Anything else relevant that doesn't fit above.
