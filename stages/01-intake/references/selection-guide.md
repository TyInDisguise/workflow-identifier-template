# Selection Guide

How to help the user pick **one** workflow from the inventory for stage
02's deep-dive. The agent helps; the agent does not choose for the user.

## Selection criteria

A good stage 02 candidate meets most of these:

1. **Runs often enough to justify documentation.** Weekly or more is
   strong. Monthly is marginal — ask whether it's worth the effort.
   Less than monthly is usually not a fit.
2. **Has real pain, volume, or strategic value.** Unpained workflows
   don't get fixed, even when documented. If nothing about the workflow
   hurts or costs meaningfully, surface that before the user commits.
3. **Specific, not umbrella.** "Qualifying inbound leads from the
   website form" is specific. "Sales process" is not. If the candidate
   is an umbrella term, ask the user to pick one slice of it.
4. **The interviewee actually performs it.** Documenting secondhand
   leads to incomplete records and missed edge cases. If the user is
   describing someone else's workflow, redirect to one they do
   themselves — or schedule the real operator to join stage 02.
5. **Scopes to roughly 30 minutes of depth elicitation.** If it's so
   broad that stage 02 would run for hours, help the user scope it
   down to a subset that fits.

## How to guide selection without choosing

The user owns the choice. The agent's job is to:

- **Present the inventory back** with the interest reason for each
  candidate visible.
- **Ask which one they want to go deep on.** Don't rank; don't steer.
- **If they're unsure**, offer two framings:
  - *"Which one would give you the most back if it worked better?"*
    (value framing)
  - *"Which one is the most annoying to do manually?"* (pain framing)
- **If they pick one that fails a criterion**, name it gently and ask:
  - "This one runs about once a quarter — is it worth the time to
    document, or would another be a better pick?"
  - "This one is pretty broad — should we pick one slice of it to focus
    on?"
- **If they pick one they don't personally perform**, ask: "Would the
  person who actually runs this be available for the stage 02
  conversation?"

## After selection

Write a **one-paragraph scope description** that includes:

- What the workflow is (in the user's words)
- Who performs it
- How often
- What the rough boundaries are — what's in scope, what's not

Present this back to the user. Ask: "Does this capture what you want to
go deep on?" Revise until they confirm before advancing to stage 02.

## When none of the candidates are a good fit

Rare, but possible. If every candidate fails criteria 1-3, name it
honestly: "Based on what you've shared, none of these may be the best
fit for deep documentation right now." Options:

- Return to the inventory pass and probe for workflows that weren't
  surfaced.
- Pick the best available candidate with the caveats noted in the
  selected-workflow output.
- Pause the session and suggest revisiting when the user has a specific
  workflow in mind.

Don't push the user into a mediocre pick for the sake of advancing.
